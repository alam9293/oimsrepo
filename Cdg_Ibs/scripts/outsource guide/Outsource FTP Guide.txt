A.  MANUAL TEST FTP via CMD


1) go to CMD

2) type    "FTP -n <Ip Address>"

	- it should say connected to <ip address>

3) type    "user <username> <password>"

	- it should say login successful


4) type    "cd <to the folder of the ftp you want>"	
	
	eg : cd folder1/folder2

	- it should say Directory successfully change  
	- if it say Failed to change directory, retry with another pattern eg : folder1/folder2 , folder1\folder2
	- note: if FOLDER GOT SPACING, Use double quote.
		eg: cd folder1/"folder test space"

5) 	create test.txt in c:/ or anywhere.. in current System.

6)	eg :type    "put c:/test.txt"   to put  c:/test.txt  to the FTP directory from (4)


7) go into ftp and check if test.txt successfully inside.

B.  Edit  FTP_OS_PRINTING.BAT    
- for TESTING USE  *make a backup of ftp_os_printing.bat


@echo off
 echo user username password > ftpcmd.dat
 
 echo cd folder1/folder2 >> ftpcmd.dat
 
 echo put %1 >> ftpcmd.dat
 
 echo quit >> ftpcmd.dat
 
 ftp -n -s:ftpcmd.dat 10.0.2.147
 del ftpcmd.dat




username = ftp username
password = ftp password
folder1/folder2  = replace with (step 4) the directory
%1       =  replace with the file location from (step 6)  
10.0.2.147 = replace with FTP IP


1) Save and Double Click on the FTP_OS_PRINTING.BAT
2) it will automatically append the username, password, go to the folder, put the file in , and close the ftp.
3) check the ftp if the file is inside successfully like (step 7)
